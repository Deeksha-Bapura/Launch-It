Build a full-stack web application called LaunchIt — a conversational AI guide that helps informal economy workers (people already making money informally) understand how to formalize their business, get the right permits, and manage basic finances. The app starts with a conversation, not a form. Scope is Houston, TX and Austin, TX only for the prototype.

STACK

Frontend: React with Vite — lives in /frontend
Backend: Node.js + Express — lives in /backend
They communicate via REST API calls

**`backend/services/permitData.js`**
Create a data file that exports two things: a `permitData` object and a `classifyBusiness` function.

The `permitData` object has two top-level keys: `houston` and `austin`. Each contains:
- `cottageFoodLaw`: object with fields `applies` (array of food types), `annualRevenueLimit` (50000), `requirements` (array of strings explaining the rules in plain language), `cost` (0), `link` (official Texas DSHS cottage food URL)
- `salesTax`: object with fields `rate` (0.0825), `threshold` (string), `filingFrequency` (string), `dueDates` (array: April 20, July 20, October 20, January 20), `registrationLink` (Texas comptroller URL)
- `generalBusinessLicense`: object with `required` (false) and `notes` explaining Houston/Austin do not require one
- `dba`: object with `required` (condition string), `cost` (25 for Houston, 24 for Austin), `filedWith` (county clerk name), `link` (county clerk URL)
- `beautyServices`: object explaining Texas Cosmetology license requirement, cost (~$50), licensing board name, and link
- `cleaningServices`: object explaining no license required but sales tax on commercial cleaning, with relevant notes
- `landscaping`: object explaining no state license for basic lawn care, but pesticide application requires separate license

The `classifyBusiness(description)` function takes a plain-language string and returns one of these strings: `'food'`, `'beauty'`, `'cleaning'`, `'landscaping'`, `'apparel'`, `'education'`, `'repair'`, `'retail'`, `'general'` — based on keyword matching against the description.

**`backend/services/documentTemplates.js`**
Create four functions and export them all:

`generateInvoice(businessName, ownerName, city)` — returns an object with type `'invoice'`, a title, an array of header fields (business name, owner name, city, invoice number placeholder, date placeholder, bill-to placeholder), an array of line item placeholders, a payment terms footer, and a practical tip about keeping copies for tax purposes.

`generateProfitLossTracker(businessType)` — returns an object with type `'profit_loss'`, a title, an income array (Sales, Other Income with zero amounts), an expenses array (Supplies/Ingredients, Packaging, Gas/Transportation, Marketing, Phone business portion, Other — all zero), and a tip explaining what profit means at tax time.

`generatePricingWorksheet(productName)` — returns an object with type `'pricing'`, a title using the product name, and a steps array walking through: ingredient/material cost per unit, packaging cost, time cost (hours × hourly rate, minimum $15/hr), overhead share, total cost (sum), markup (suggested 2-3×), and final suggested selling price. Include a warning that many informal business owners undercharge because they forget to count their own time.

`generateSocialPost(businessName, product, city)` — returns an object with type `'social_post'`, a title, and a `platforms` object with three keys: `instagram` (a template post with emoji, product, city, DM to order, hashtags), `facebook` (a friendly neighbor-focused template), and `nextdoor` (a short local business announcement template).

- Validate that `messages` exists in the request body and is an array, returning 400 if not

- Pass errors to the next middleware
